About this app

This app is a quick demo showing how to make a single page web app using AngularJS. If you would like to clone it, the repo is available on github.

xkcd is a long running webcomic by Randall Munroe but I suspect if you are reading this, xkcd is not new to you.

How does it work?

Most importantly, this is a single page web app. The resource in the address bar at the top is index.html and that does not change throughout the use of the app. All of the magic that makes it so is provided by AngularJS. One of the first decisions to make in a single page web app I discovered is whether you want to support deep linking or not. Deep linking is the ability to be able to use a URL to be able to jump straight to a part deep inside the app. I decided that I did want to support deep linking so I used ui-router to structure my app. In addition to provide a good way to structure your app, ui-router offers deep linking through fragments following the # end of a URL. A hash prefix (such as a !) can be added as well if the fragment confuses your web server. Angular can use the HTML 5 history API if available and fall back to hashbangs if not. This model keeps the trusty back button working as well.

Angular allows you to structure your app into small (testable) chunks spread across different files which has obvious development and testing benefits. Angular also offers dependency injection to wire it altogether. Angular supports MVW (Model View Whatever) framework. In this case I am using a MVVM (Model View ViewModel) framework as the $scope and controller work togther whislt keeping the mark-up and model separate.

Accessing the comics?

The xkcd site offers a JSON interface which is very handy. Unfortunately it does not support CORS so there is no loading the data strainght into the app. Fortunately the Angular $resource service offers a JSONP alternative. JSONP can help you get around CORS issues by generating a script tag in the DOM which the browser will dutifully load for you from any domain. To allow your app to access the data the data has to come back from the remote server wrapped up in a function call which will execute and inject the data into your app. Obviously you have to trust the source (or be prepared to filter the response) as the remote server is injecting data directly into your app. Fortunately the xkcd site offers a JSONP service which has been used in this app. Angular will dynamically generate a JavaScript method with an appropriate namespace that the remote server wraps the data in automatically.

What about testing?

Automated tests should be a part of any software and the same goes for web apps like this one. I've used Jasmine for testing the ComicController which is the main controller in my app. You can do server side testing using Node and Karma but for this app, I've kept it simple and developed a small suite of tests that run in the browser.

What AngularJS technologies were used?

The following AngularJS technologies were used in this app:
  • AngularUI (UI-Bootstrap) which provides Twitter Bootstrap written natively in AngularJS
  • ui-router which provides flexible routing and deep linking
  • ngResource which provides interaction support with RESTful services
  • ngAnimate which provides support for JavaScript, CSS3 transition and CSS3 keyframe animation
  • Jasmine which is a behavior-driven development framework for testing JavaScript code
Other interesting AngularJS technologies used in this app:
  • $q service which is a promise/deferred implementation
  • Unit Testingt which how Angular apps can be tested
